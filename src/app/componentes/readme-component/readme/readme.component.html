Please take in mind that if you use offline version, you can calculate the difference of days between the two dates because it is not needed to go online to perform that task; but you could not retrieve any data from the API and hence when you want to see historic facts happened on those days and months, an error which prompts you to check your connection would be thrown.

I fixed a bug from the old non PWA version where error messages and proper data could be shown together.

WHAT DOES Datepedia-pwa DO?

You can set two different dates, then calculate how many days have passed between them. But also you will be able to know what happenned on such days and month through History thanks to an API

WHAT HAVE I LEARNED OR IMPLEMENTED WHILE DEVELOPING Datepedia-pwa?

This is a little list of things I implemented or done on this project

  • Create and install an angular Progressive Web App. Also learned about the concept of service workers, web workers, webmanifest.
  • I used reactive forms -until now, I was more used to template driven ones-, with built-in and custom validators. For example, custom validator is created to ensure, if user inputs same exact date, form will be invalid.
  • You can enter the dates in the order you want. No choronological order required. For example you can set 24/06/2021 on the first box and 06/07/2009 on the second. It was kind of logic challenge to control it and make usability a little better than forcing the user to set ordered dates or showing negative results, like I have seen on similar websites.
  • I used an external API (visit here) to get the events occurred on each date. I managed to get 5 random results from the api for each date, also ensuring they can´t be repeated.
  • To get the results of those API, the async HTTP requests were made with Angular´s httpClient get() method. It returns an Observable so I had to apply some Reactive Programming skills. As it also involves async requests, I used promises to manage results, to ensure I obtain and control data flow the way I need. To obtain the random 5 results mentioned above I used vanilla JS methods like Math.random(), because RXJS operators like take don´t work here. Reason is, the events are emitted by the API all of them together, to be able to use the take() operator they should be emitted by the Observable one by one.
  • I also provide a branch on GitHub with the same code below, but using async/await to see the differences. Async await provides a cleaner code since the behavior is more similar to what we see on synchronous code
  • Used interfaces to create custom data types to be manage the results shown to the user, it makes data retrieved by the API easier to manipulate and ensures integrity and maintenance. Also used an interface to create a custom error object type, and to manage error messages, if something if wrong with the API for example.
  • Used basic angular animations to flip the box with the results and the button to show events ocurred on each date.
  • Basic testing with karma and jasmine, maybe learn Protactor in the future if I find easier and faster to automate.
  • Form styles found here.
  • Customized 404 error page. Used code available here.
  • Other libraries used are Bootstrap and Font Awesome.
